# 智能物流系统 UI 演示模式

## 概述
此模式允许你在没有 NVIDIA Jetson 硬件和 ROS 环境的情况下运行和查看智能物流系统的用户界面。

## 修改说明

### 1. GPIO 模拟 (`utils/gpio.py`)
- 添加了 `MockGPIO` 类来模拟 Jetson.GPIO 模块
- 当检测不到 Jetson.GPIO 时自动切换到模拟模式
- 所有 GPIO 操作将被记录但不会实际执行

### 2. ROS 命令模拟 (`functional/roslaunch.py`)
- 添加了对 Windows 系统和模拟模式的检测
- ROS 命令（roslaunch, rosrun）将打印日志而不是实际执行
- 所有功能调用都添加了异常处理

## 安装步骤

### Windows 系统

1. 安装 Python 3.7 或更高版本

2. 安装依赖包：
```bash
pip install -r requirements_windows.txt
```

3. 运行 UI 演示：
```bash
python run_ui_demo.py
```

或者直接运行：
```bash
python main.py
```

## 功能说明

### 可用功能
- ✅ 查看完整的 UI 界面布局
- ✅ 测试所有按钮的点击响应
- ✅ 查看日志输出
- ✅ 切换语言（中文/英文）
- ✅ 测试界面交互逻辑

### 受限功能
- ❌ 实际的硬件控制（雷达、导航、吸泵等）
- ❌ ROS 节点启动和通信
- ❌ 摄像头实时画面（除非你有兼容的摄像头）
- ❌ 机械臂控制
- ❌ AGV 运动控制

## 界面功能模块

### 1. 顶部标题栏
- 系统标题显示
- 语言选择（中文/英文）

### 2. 左侧控制面板

#### 智慧物流任务
- **快速启动**：一键启动完整的物流任务流程

#### 基础控制
- **激光雷达**：开启/关闭雷达
- **导航系统**：开启/关闭导航

#### 物流任务
- **移动到货架**：导航到指定货架位置
- **循环分拣**：执行循环分拣任务
- **自动充电**：自动导航到充电桩

#### 视觉功能
- **OCR 文字识别**：识别文字信息
- **ARUCO 码识别**：识别 ARUCO 标记
- **QR 码识别**：识别二维码

#### 运动控制
- **手柄控制**：使用游戏手柄控制 AGV
- **键盘控制**：使用键盘控制 AGV

### 3. 右侧显示区域

#### 摄像头视图
- 机械臂摄像头视图
- AGV 2D 摄像头视图

#### 日志浏览器
- 实时显示系统日志
- 日志清除按钮

## 模拟模式行为

当你点击各个功能按钮时：

1. **雷达控制**：会在控制台打印 `[MOCK MODE] Radar launch simulated`
2. **导航控制**：会在控制台打印 `[MOCK MODE] Navigation simulated`
3. **其他功能**：类似地会打印模拟信息

所有操作都会在日志浏览器中显示相应的消息。

## 故障排除

### 问题：缺少 PyQt5
```bash
pip install PyQt5
```

### 问题：缺少 OpenCV
```bash
pip install opencv-python
```

### 问题：界面显示异常
- 确保你的显示器分辨率至少为 1280x800
- 尝试调整窗口大小

### 问题：中文显示乱码
- 确保系统支持中文字体
- 字体文件位于 `resources/font/SIMFANG.TTF`

## 注意事项

1. 此模式仅用于 UI 展示和测试，不能用于实际的物流控制
2. 某些功能可能会因为缺少硬件而无法完全展示
3. 在实际的 Jetson 设备上运行时，会自动使用真实的硬件接口

## 恢复到生产模式

如果你需要在 Jetson 设备上运行实际系统：

1. 确保已安装 Jetson.GPIO：
```bash
pip install Jetson.GPIO
```

2. 确保 ROS 环境已正确配置

3. 直接运行：
```bash
python main.py
```

系统会自动检测并使用真实的硬件接口。
