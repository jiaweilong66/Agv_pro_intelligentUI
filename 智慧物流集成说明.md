# 智慧物流完整流程集成说明

## 概述

已将 `~/Smart_Logistics_Kit-22-map/` 中的完整智慧物流工作流程集成到快速启动界面中。点击"快速启动"按钮后，将自动执行包括导航、视觉抓取和分拣在内的完整工作流程。

## 工作流程

### 执行步骤

快速启动按钮现在会触发 `SmartLogisticsProcess`，它会依次执行三个主要步骤:

1. **启动 myAGV 里程计系统**
   ```bash
   roslaunch myagv_odometry myagv_active.launch
   ```

2. **启动导航系统**
   ```bash
   roslaunch myagv_navigation simplify_logistics_navigation_active.launch
   ```

3. **执行智慧物流主程序**
   ```bash
   cd ~/Smart_Logistics_Kit-22-map/
   python3 main.py
   ```

### 任务进度可视化

快速启动界面底部的5个任务进度节点现在映射到实际的工作流程阶段:

- **节点 1: "前往装货区"** - 里程计系统已启动，myAGV 准备就绪
- **节点 2: "拾取货物"** - 导航系统激活，准备拾取
- **节点 3: "前往卸货区"** - 智慧物流程序已启动，正在导航
- **节点 4: "卸载货物"** - 270机械臂正在执行视觉抓取和分拣
- **节点 5: "成功"** - 完整工作流程成功完成

每个节点根据状态改变颜色:
- **蓝色 (...)**: 等待中/进行中
- **绿色 (✓)**: 成功完成
- **红色 (✗)**: 发生错误

## 使用说明

### 前置条件

1. **ROS 环境**: 确保 ROS 已正确安装并配置
   ```bash
   source /opt/ros/melodic/setup.bash  # 或您的 ROS 版本
   ```

2. **智慧物流目录**: 目录必须存在于:
   ```
   ~/Smart_Logistics_Kit-22-map/
   ```

3. **ROS Master**: 在启动 UI 之前启动 roscore:
   ```bash
   roscore
   ```
   
   或使用提供的自动启动 roscore 的脚本:
   ```bash
   ./start_with_ros.sh
   ```

### 运行工作流程

1. **启动 UI**:
   ```bash
   ./start_ui.sh
   ```

2. **点击"快速启动"按钮** (在快速启动界面中)

3. **监控进度**:
   - 观察5个任务节点随着工作流程进展而改变颜色
   - 在日志输出区域阅读详细的日志消息
   - 每个步骤都清楚地标记有 [步骤 X/5] 标记

4. **如需停止**:
   - 点击"停止工作流程"按钮终止
   - 所有进程将被干净地关闭

### 预期行为

当工作流程成功运行时:

1. **初始化** (5秒)
   - 里程计系统启动
   - 节点1变绿

2. **导航设置** (10秒)
   - 导航系统启动
   - 节点2变绿

3. **智慧物流执行** (时间可变)
   - 主程序启动
   - myAGV 导航到指定位置
   - 270机械臂执行基于视觉的物体检测
   - 物体被拾取和分拣
   - 节点3、4、5随着任务完成而变绿

4. **完成**
   - 所有节点显示绿色对勾
   - 显示成功消息
   - 所有进程干净终止

## 故障排除

### 问题: "ROS节点初始化错误"

**解决方案**: 在启动 UI 之前启动 roscore
```bash
# 终端 1
roscore

# 终端 2
./start_ui.sh
```

### 问题: "找不到智慧物流目录"

**解决方案**: 验证目录是否存在
```bash
ls ~/Smart_Logistics_Kit-22-map/
```

如果缺失，将 Smart_Logistics_Kit-22-map 目录克隆或复制到您的主文件夹。

### 问题: "找不到 main.py"

**解决方案**: 检查 main.py 是否存在于智慧物流目录中
```bash
ls ~/Smart_Logistics_Kit-22-map/main.py
```

### 问题: 工作流程在特定节点停止

**检查**:
1. ROS 启动文件已正确安装
2. 导航地图可用
3. 摄像头设备可访问
4. 机械臂已连接

**查看详细错误** 在 UI 的日志输出区域。

### 问题: 进程无法干净终止

**解决方案**: 工作流程包含自动清理，但如果需要:
```bash
# 杀死所有 ROS 节点
rosnode kill -a

# 或重启 roscore
killall -9 roscore rosmaster
roscore
```
