# 摄像头冲突解决方案

## 问题描述

运行Smart Logistics时出现错误：
```
Exception: 无法打开摄像头
```

## 原因分析

1. **UI启动时打开了摄像头**
   - AGV 2D Camera (video0)
   - Arm Camera (video1)

2. **Smart Logistics也需要摄像头**
   - main.py需要打开同样的摄像头设备

3. **冲突**
   - 同一个摄像头设备不能被两个程序同时使用
   - Linux系统中，摄像头设备是独占的

## 解决方案

### 自动释放和恢复

快速启动现在会自动处理摄像头：

1. **启动Smart Logistics前**
   - 自动停止UI的摄像头线程
   - 释放 /dev/video0 和 /dev/video1
   - 等待1秒确保完全释放

2. **Smart Logistics运行中**
   - main.py可以独占使用摄像头
   - 进行视觉识别和抓取

3. **Smart Logistics完成后**
   - 自动重启UI的摄像头
   - 恢复摄像头显示

## 技术细节

### 释放摄像头

```python
# 停止摄像头线程
camera_capture.deactivate_middleware()  # 停止中间件
camera_capture.stopped()                 # 发送停止信号
camera_capture.wait(1000)                # 等待线程结束
if camera_capture.isRunning():
    camera_capture.terminate()           # 强制终止
```

### 等待释放

```python
import time
time.sleep(1)  # 等待1秒确保设备完全释放
```

### 重启摄像头

```python
# Smart Logistics完成后自动重启
self.agv_camera_capture = self.start_camera_capture(...)
self.arm_camera_capture = self.start_camera_capture(...)
```

## 常见问题

### Q: 为什么摄像头画面消失了？

A: 这是正常的！Smart Logistics需要独占使用摄像头。完成后会自动恢复。

### Q: 如果摄像头没有恢复怎么办？

A: 
1. 点击"停止"按钮
2. 关闭UI
3. 重新启动UI

### Q: 可以在Smart Logistics运行时看到摄像头画面吗？

A: 不能。摄像头设备是独占的，只能被一个程序使用。但你可以在日志中看到识别结果。

### Q: 如果不想释放摄像头怎么办？

A: 那就不能使用快速启动。你需要：
1. 手动启动雷达和导航
2. 关闭UI
3. 在终端运行 `python3 main.py`
