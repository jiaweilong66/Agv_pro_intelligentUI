# 智慧物流快速启动 - 最终使用说明

## 功能说明

快速启动现在会自动完成所有步骤：

1. ✅ 启动雷达（Lidar）系统 → 按钮变红显示"关闭"
2. ✅ 启动导航（Navigation）系统 → 按钮变红显示"关闭"
3. ✅ 在后台线程运行 `Smart_Logistics_Kit-22-map/main.py`
4. ✅ UI保持响应，不会卡死
5. ✅ 实时显示main.py的输出

## 使用步骤

### 1. 启动UI

```bash
cd ~/intelligent-logistics-system
./start_ui.sh
```

### 2. 点击快速启动

1. 在主菜单点击"快速启动"
2. 点击左上角的"快速启动"按钮
3. 等待系统自动完成所有步骤

### 3. 观察进度

观察5个任务节点的变化：

- **节点1（蓝色→绿色）**: 雷达系统启动完成
- **节点2（蓝色→绿色）**: 导航系统启动完成
- **节点3（蓝色→绿色）**: Smart Logistics程序启动
- **节点4（蓝色→绿色）**: myAGV导航中，270机械臂工作中
- **节点5（蓝色→绿色）**: 所有任务完成！

### 4. 监控日志

在日志输出区域可以看到：
- 系统启动信息
- main.py的实时输出（带 `[MAIN.PY]` 标记）
- 任务进度更新
- 错误和警告信息

### 5. 停止工作流程

点击"停止"按钮会：
- 终止main.py进程
- 关闭导航系统
- 关闭雷达系统
- 重置所有节点

## 技术细节

### 为什么UI不会卡死？

`SmartLogisticsProcess` 在 `QThread` 中运行：
- 独立线程执行main.py
- UI线程保持响应
- 通过信号传递消息到UI

### main.py如何运行？

```python
# 在子进程中运行
subprocess.Popen(
    ["python3", "main.py"],
    cwd="~/Smart_Logistics_Kit-22-map/",
    stdout=subprocess.PIPE,  # 捕获输出
    stderr=subprocess.STDOUT,
    universal_newlines=True
)
```

### 输出如何显示？

```python
# 逐行读取输出
while process.poll() is None:
    line = process.stdout.readline()
    if line:
        self.notify_published(f"[MAIN.PY] {line.strip()}")
```

## 常见问题

### Q: UI会卡死吗？

A: 不会！main.py在独立的QThread中运行，UI始终保持响应。

### Q: 可以看到main.py的输出吗？

A: 可以！所有输出都会实时显示在日志区域，带有 `[MAIN.PY]` 标记。

### Q: 如何停止工作流程？

A: 点击"停止"按钮，会自动终止main.py并关闭雷达和导航。

### Q: 雷达和导航按钮会更新吗？

A: 会！启动后按钮会变红显示"关闭"状态。

### Q: 可以手动关闭雷达或导航吗？

A: 可以，但不建议在工作流程运行时关闭。最好使用"停止"按钮。

### Q: 如果main.py出错怎么办？

A: 
1. 查看日志输出中的 `[MAIN.PY]` 消息
2. 查看 `[ERROR]` 消息
3. 对应的节点会变红
4. 点击"停止"按钮清理

### Q: 可以运行多次吗？

A: 可以！完成后点击"停止"清理，然后可以再次点击"快速启动"。
