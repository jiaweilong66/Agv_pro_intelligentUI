# ROS节点冲突修复说明

## 问题描述

你遇到的错误是：

```
rospy.exceptions.ROSException: rospy.init_node() has already been called with different arguments
```

## 原因

这个错误发生是因为：
1. UI启动时已经初始化了一个ROS节点（名为 `intelligent_logistics_ui`）
2. 当你点击"移动到货架"按钮时，`MapNavigation` 类尝试初始化另一个节点（名为 `map_navigation`）
3. ROS不允许在同一个进程中多次调用 `rospy.init_node()`

## 已应用的修复

### 1. 修复了MapNavigation类

修改了 `functional/depend/navigation.py`，现在会检查ROS节点是否已经初始化：

```python
# 检查ROS节点是否已经初始化
try:
    rospy.get_node_uri()
    # 节点已初始化，跳过init_node
except rospy.exceptions.ROSException:
    # 节点未初始化，进行初始化
    rospy.init_node('map_navigation', anonymous=False)
```

### 2. 添加了工作流程保护

修改了 `on_move_to_shelf()` 和 `on_circular_sorting()` 函数：
- 这些函数现在会检查智慧物流工作流程是否正在运行
- 如果正在运行，会显示警告并阻止执行
- 这样可以避免ROS节点冲突

## 为什么会发生这个问题

### ROS节点初始化规则

在ROS中，每个Python进程只能调用一次 `rospy.init_node()`。如果尝试用不同的参数再次调用，ROS会抛出异常。

### 我们的架构

```
UI进程 (intelligent_logistics_ui节点)
    ├── 快速启动工作流程
    │   └── 启动子进程（独立进程，无冲突）
    │       └── Smart_Logistics_Kit-22-map/main.py
    │           └── 可以初始化自己的ROS节点
    │
    └── 单独功能（同一进程）
        ├── 移动到货架
        │   └── 使用MapNavigation（需要ROS节点）
        └── 循环分拣
            └── 使用MapNavigation（需要ROS节点）
```

### 修复方案

1. **MapNavigation** 现在会在初始化前检查节点是否存在
2. **单独功能** 在工作流程运行时被阻止
3. **子进程隔离** - 智慧物流在独立进程中运行

## 总结

ROS节点冲突已通过以下方式修复：
1. MapNavigation现在会检查现有节点
2. 工作流程运行时阻止单独功能

## 常见问题

**Q: 为什么不能同时使用快速启动和单独功能？**
A: 因为它们都需要初始化ROS节点，而ROS不允许在同一进程中多次初始化。

**Q: 如果我需要在工作流程中途使用其他功能怎么办？**
A: 先停止工作流程，然后使用单独功能。

**Q: 修复后还会有错误吗？**
A: 如果按照指南使用（不混合使用），就不会有错误。

**Q: 我可以重启工作流程吗？**
A: 可以。等待当前工作流程完成或停止，然后可以再次点击"快速启动"。
