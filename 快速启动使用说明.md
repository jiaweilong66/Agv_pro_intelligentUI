# 快速启动使用说明

## 概述

快速启动功能已经重新设计，现在它会：
1. 自动启动雷达（Lidar）系统
2. 自动启动导航（Navigation）系统
3. 准备好系统，等待你运行Smart Logistics主程序

## 为什么这样设计？

之前的设计试图在UI中自动运行 `Smart_Logistics_Kit-22-map/main.py`，但这会导致：
- UI卡死（因为main.py是一个长时间运行的程序）
- ROS节点冲突
- 无法看到main.py的实时输出

新设计让你可以：
- ✅ UI保持响应，不会卡死
- ✅ 在独立终端中看到main.py的完整输出
- ✅ 更容易调试和监控
- ✅ 可以随时停止main.py而不影响UI

## 使用步骤

### 方法1：使用UI + 终端（推荐）

1. **启动UI**
   ```bash
   cd ~/intelligent-logistics-system
   ./start_ui.sh
   ```

2. **在UI中点击"快速启动"**
   - 在主菜单点击"快速启动"
   - 点击左上角的"快速启动"按钮
   - 等待系统准备完成（节点1、2、3变绿）

3. **在新终端中运行Smart Logistics**
   ```bash
   # 打开新终端（Ctrl+Alt+T）
   cd ~/Smart_Logistics_Kit-22-map/
   python3 main.py
   ```

4. **监控进度**
   - 在UI的日志输出区域查看系统状态
   - 在终端中查看main.py的详细输出
   - 观察myAGV导航和270机械臂工作

### 方法2：使用辅助脚本

1. **启动UI并点击快速启动**（同方法1的步骤1-2）

2. **使用辅助脚本运行Smart Logistics**
   ```bash
   # 打开新终端
   cd ~/intelligent-logistics-system
   ./run_smart_logistics.sh
   ```

   这个脚本会：
   - 检查Smart Logistics目录是否存在
   - 检查roscore是否运行
   - 自动切换到正确的目录
   - 运行main.py

## 任务节点说明

快速启动后，你会看到5个任务节点：

### 节点1-3：由快速启动自动完成

- **节点1: "前往装货区"** 
  - 状态：雷达系统启动
  - 自动完成：✓

- **节点2: "拾取货物"**
  - 状态：导航系统启动
  - 自动完成：✓

- **节点3: "前往卸货区"**
  - 状态：系统准备就绪
  - 自动完成：✓

### 节点4-5：由Smart Logistics main.py完成

- **节点4: "卸载货物"**
  - 状态：等待main.py运行
  - 当myAGV开始导航时，手动更新为完成

- **节点5: "成功"**
  - 状态：等待main.py完成
  - 当整个流程完成时，手动更新为完成

## 停止工作流程

### 停止UI部分

在UI中点击"停止"按钮：
- 关闭导航系统
- 关闭雷达系统
- 重置任务节点

### 停止main.py

在运行main.py的终端中：
- 按 `Ctrl+C` 停止程序

## 常见问题

### Q: 为什么不能在UI中直接运行完整流程？

A: 因为Smart Logistics的main.py是一个长时间运行的程序（可能需要5-30分钟），如果在UI进程中运行会导致UI卡死无响应。分离运行可以保持UI响应性。

### Q: 我必须手动运行main.py吗？

A: 是的，这是最稳定和可靠的方式。你可以：
- 在终端中看到完整的输出
- 随时用Ctrl+C停止
- 不会影响UI的响应性

### Q: 节点4和5会自动更新吗？

A: 目前不会自动更新。它们显示为"等待"状态，提醒你需要运行main.py。未来版本可能会添加自动监控功能。

### Q: 可以不用快速启动，直接运行main.py吗？

A: 可以，但你需要：
1. 手动启动roscore
2. 手动启动雷达
3. 手动启动导航
4. 然后运行main.py

快速启动帮你自动完成前3步。

### Q: 如果main.py运行出错怎么办？

A: 
1. 查看终端中的错误信息
2. 检查硬件连接（摄像头、机械臂）
3. 检查ROS节点状态：`rosnode list`
4. 如果需要，停止并重新开始

## 优势

这种设计的优势：

1. **UI不会卡死** - UI始终保持响应
2. **完整输出** - 可以看到main.py的所有输出
3. **易于调试** - 错误信息直接显示在终端
4. **灵活控制** - 可以随时停止和重启main.py
5. **独立运行** - main.py在独立进程中，不会影响UI
