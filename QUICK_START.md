# 快速开始 - Windows UI 演示

## 一键启动（推荐）

### 方法 1：使用批处理文件

1. 双击运行 `install_windows.bat` 安装依赖
2. 双击运行 `test_imports.py` 测试依赖是否安装成功（可选）
3. 双击运行 `run_ui_demo.bat` 启动 UI

### 方法 2：使用命令行

```bash
# 1. 安装依赖
pip install -r requirements_windows.txt

# 2. 测试依赖（可选）
python test_imports.py

# 3. 运行 UI
python run_ui_demo.py
```

## 已知问题和解决方案

### 问题 1: 缺少 paddle 模块
这是正常的！PaddlePaddle（用于 OCR 识别）在 Windows 上安装较复杂，已设为可选。
- OCR 功能会显示"OCR Not Available (Mock Mode)"
- 其他功能不受影响

### 问题 2: 缺少 rospy 模块
这是正常的！ROS 只在 Linux 系统上可用。
- 所有 ROS 相关功能会显示 `[MOCK MODE]` 标记
- UI 界面可以正常显示和交互

### 问题 3: 缺少 pymycobot 模块
这是正常的！机械臂控制库只在实际硬件上需要。
- 手柄控制功能会在模拟模式下运行
- 不影响 UI 显示

## 测试功能

### 1. 语言切换
- 点击右上角的语言选择框
- 选择"中文"或"English"
- 界面文字会立即切换

### 2. 快速启动
- 点击"快速启动"按钮
- 观察日志输出区域的消息
- 在模拟模式下会显示 `[MOCK MODE]` 标记

### 3. 各个功能按钮
- 点击任意功能按钮
- 查看控制台和日志输出
- 所有操作都会被记录

### 4. 日志管理
- 观察日志输出区域的实时消息
- 点击"清除日志"按钮清空日志

## 常见问题

### Q: 提示缺少模块怎么办？
A: 运行 `pip install -r requirements_windows.txt`

### Q: 界面显示不完整？
A: 调整窗口大小或检查屏幕分辨率（建议至少 1280x800）

### Q: 中文显示乱码？
A: 确保系统支持中文，字体文件在 `resources/font/SIMFANG.TTF`

### Q: 点击按钮没反应？
A: 这是正常的，因为没有实际硬件。查看控制台输出可以看到模拟信息。

### Q: 摄像头区域是空白的？
A: 这是正常的，因为没有连接 Jetson 摄像头。
