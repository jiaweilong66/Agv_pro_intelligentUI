# 快速启动修复总结

## 修复的问题

### 问题1：雷达和导航按钮状态没有更新
**修复：** 快速启动现在会调用 `on_lidar_toggle()` 和 `on_navigation_toggle()`，这会自动更新按钮状态为"关闭"（红色）。

### 问题2：程序卡死
**原因：** 之前的设计试图在子进程中运行 `Smart_Logistics_Kit-22-map/main.py`，但这个程序需要长时间运行（5-30分钟），导致UI等待子进程完成而卡死。

**修复：** 改变设计思路：
- 快速启动只负责准备系统（启动雷达和导航）
- 用户在独立终端中手动运行 `main.py`
- UI保持响应，不会卡死

### 问题3：无法到达抓取盒子的位置
**原因：** 程序卡死导致后续流程无法执行。

**修复：** 现在 `main.py` 在独立进程中运行，可以正常执行完整流程。

## 故障排除

### 如果雷达或导航启动失败

**检查：**
```bash
# 检查roscore是否运行
ps aux | grep roscore

# 检查ROS节点
rosnode list

# 查看ROS日志
tail -f ~/.ros/log/latest/rosout.log
```

**解决：**
1. 确保roscore正在运行
2. 重启UI
3. 再次尝试快速启动

### 如果main.py运行出错

**检查：**
```bash
# 检查目录是否存在
ls ~/Smart_Logistics_Kit-22-map/main.py

# 检查设备
ls /dev/video*
ls /dev/ttyACM*

# 检查ROS话题
rostopic list
```

**解决：**
1. 确保所有硬件已连接
2. 确保雷达和导航已启动
3. 查看终端中的错误信息
4. 根据错误信息调试
